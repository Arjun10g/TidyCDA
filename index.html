Why use Tidy Categorical Data Analysis?

  • Consistency: The packages in Tidyverse share a consistent design philosophy and syntax, which makes it easier to learn and use multiple packages.
  • Data manipulation: Tidyverse packages such as dplyr and tidyr provide powerful tools for data manipulation, such as filtering, grouping, summarizing, and reshaping data.
  • Data visualization: Tidyverse packages such as ggplot2 provide powerful tools for data visualization, allowing you to create complex and customizable plots with ease.
  • Data import and export: Tidyverse packages such as readr and writr provide easy-to-use functions for importing and exporting data from various file formats.
  • Pipe operator: Tidyverse introduces the `%>%` pipe operator, which allows you to chain together multiple operations in a concise and readable way.
  • Data science workflow: Tidyverse provides a consistent and efficient workflow for data science tasks, from data cleaning and manipulation to visualization and modeling.

Benefits of using %>% operator

  • Readability: The piping operator helps to make your code more readable by breaking down complex operations into a series of smaller, more manageable steps. This makes it easier to understand what each step is doing and how it fits into the overall process.
  • Flexibility: The piping operator allows you to easily modify and extend your code without having to rewrite the entire process. You can add or remove steps as needed, or modify existing steps to accommodate changes in your data or analysis requirements.
  • Efficiency: By chaining together multiple operations, you can avoid the need to create intermediate variables, which can be time-consuming and resource-intensive. This can make your code more efficient and reduce the likelihood of errors.
  • Consistency: The piping operator encourages a consistent coding style and helps to enforce best practices in data analysis. By breaking down complex operations into a series of smaller steps, you can ensure that each step is clear, concise, and well-documented.

In this example, we can see how chaining operations can look much more complex in base R compared to tidyverse. Additionally, we can make relevant changes to the data and chain operations much more easily. Various graphical frameworks use similar additive formats, that allow you to chain operations or add layers.

Working with Tables in R requires understanding whether the data is in case form or frequency form. First, we begin by converting the dataframe to a tibble, since the data is currently in case form.

Case form (long form) represents each observation as a separate row in a data frame. Frequency form (wide form) represents the counts of values in a table. Case form is used for individual observations, while frequency form is used for counting values. The choice of form depends on the analysis being performed.

The code below uses the piping operator (%>%) to pass the tibble as the data argument to the xtabs() function where (.) is used as a placeholder for the left-hand operator. The xtabs() function creates a cross-tabulation table with counts of observations for each combination of the “Sex” and “Improved” variables. We can additionally, specify a third variable to stratify Sex and Improved over another variable. If the data is in case form, we do not need to specify the left hand side of the (~) operator.

Arthritis <- Arthritis %>% as_tibble
        
Arthritis %>% xtabs(~Sex + Improved,data = .)
##         Improved
## Sex      None Some Marked
##   Female   25   12     22
##   Male     17    2      6
my_table <- Arthritis %>% xtabs(~Sex + Improved + Treatment,data = .)

Since, the data is now in tabular format, in the form of an array. We can index the rows, columns and layers using the square brackets to access layers of interest.

Converting Tables to Proportions can be extremely helpful in certain cases. The first operation is the prop.table() function, which creates a table of proportions for a given table. The margin argument specifies which margins (rows, columns or layers) to use for the calculation of proportions, with margin = c(1,3) indicating that the proportions should be calculated across the rows and the layer dimension of the table.

The second operation is the round() function, which rounds the proportion values to 2 decimal places.

my_table %>% prop.table(margin = c(1,3)) %>% round(2)

Other helpful ways to deal with factors involved simple ways of ordering factors. In the first case, we order factors by Frequency, since there are more females in the dataset compared to males, they are ordered as such. The pull functions works similarly to Arthritis$Sex and pulls the column out as a vector, fct_count() provides the counts for each factor.

In the second case, we order the factor based order in which they first appear.

In the third case, we convert Improved to numeric and order them based on numeric values.

There are several methods that allow users to create ordered factors for plots. If we wish to reverse, the order of the plot or order it based of other values of interest, we can use fct_rev() and fct_reorder(). fct_rev() reverses the factor, wherease fct_reorder() allows users to reorder a factor based on the values of another variable.

Punishment %>% ggplot(aes(education, Freq)) + 
  geom_bar(stat = 'identity', fill = 'red', alpha = 0.6)

Punishment %>% mutate(education = education %>% fct_rev ) %>% ggplot(aes(education, Freq)) + 
  geom_bar(stat = 'identity', fill = 'red', alpha = 0.6)

Punishment %>% mutate(education = education %>% fct_reorder(Freq)) %>% ggplot(aes(education, Freq)) + 
  geom_bar(stat = 'identity', fill = 'red', alpha = 0.6)

Saving Tables as Images

To Publish Tables, there are several ways to go. In this tutorial, tables are produced as images. A custom function is written below that utilizes the kableExtra Package to produce tables in html format. The janitor package provides tidy version of tables.

library(janitor)
## 
## Attaching package: 'janitor'
## The following objects are masked from 'package:stats':
## 
##     chisq.test, fisher.test
apa_table_function <- function(table,tablename,title) {
  require(kableExtra)
  dat <- table
  rows <- nrow(dat)
  kbl(dat, booktabs = T,digits = 2, caption = paste0("<span style = 'color:black;padding-left:1.32cm;  margin:0px'><strong><b>",tablename,"</b></strong></span> <br>","<span style = 'color:black;padding-left:1.32cm;;margin: 0px'><i>",title,"</i></span>"),align = "c", escape = FALSE) %>% 
  kable_styling(font_size = unit(12, 'pt'), html_font = "Times New Roman") %>% 
    row_spec(0, extra_css = "text-align: center;border: 1px black; border-style: solid none solid none") %>% 
    row_spec(2:rows, extra_css = "text-align: center;border: 1px black; border-style: none none none none") %>% 
    row_spec(rows, extra_css = "text-align: center;border: 1px black; border-style: none none solid none")
}

For simple tables, the function has easy implementation.

HairEyeColor %>% as.data.frame() %>% expand.dft() %>% tabyl(Hair,Sex) %>% apa_table_function('Table 1','Hair Color by Sex')

For more complex tables, you can utilize helper functions such as pack_rows in kableExtra to provide additional functionality. Since, in this case we obtain a list with each list element containing five rows.

We can either use the map function to iterate over each list item and produce two plots for Males and Females.

HairEyeColor %>% as.data.frame() %>% expand.dft() %>% tabyl(Hair,Eye,Sex) %>% map2(c('Females', ' Males'),~apa_table_function(.x,'Table 2',paste0('Hair color and Eye color for', .y)))

Or, we can utilize pack_rows() to combine the two into a single table. However, this method utilizes more intricate knowledge of the kableExtra package.

HairEyeColor %>% as.data.frame() %>% expand.dft() %>% tabyl(Hair,Eye,Sex) %>%
  rlist::list.rbind() %>% remove_rownames() %>% apa_table_function('Table 3', 'Hair and Eye Color by Sex') %>% pack_rows('Female', 1,4,label_row_css = 'border-bottom: 1px solid black; text-indent: 1.3cm;') %>% pack_rows('Male', 5,8,label_row_css = 'border-top: 1px solid black;border-bottom: 1px solid black; text-indent: 1.3cm;') %>% row_spec(8, extra_css = 'border-bottom: none')
